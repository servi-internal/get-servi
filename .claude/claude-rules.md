# Claude Rules

All rules configured for this project / Claude AI.

---

## Chat & summaries

- Do not write summaries in the chat or create markdown files after making updates. Just say "Done!"
- Do not create .md summary files. Give the summary in the chat and keep it very short unless the user asks for more.
- Do not create markdown files after implementation.

---

## Code style

- Never use emoji in code (only in readme.md). Use React, Lucide, or Bootstrap Icons instead.
- Always keep file names in all lowercase.

---

## Python

- Use `pip3` or `python3` in Python projects.

---

## Environment files

- Read `.env.local` and `.env.production` using the `cat` command.
- Modify them using the `echo` command in the terminal.

---

## Commands

- Do not allow the command: `npm run dev`.

---

## Quality

- Always fix linters first.

---

## Web search

- Always search the web for year 2026 (when year is relevant).

---

## Kickstart (project review)

When asked to kickstart:

1. Review the various folders in the workspace.
2. Identify which folders are frontend and which are backend.
3. Describe major high-level architectural characteristics for each folder/program and what the combined product is and its core business value.
4. List major screens (routes) and data elements from a non-technical product/business point of view.
5. Describe what the user can do with the program.
